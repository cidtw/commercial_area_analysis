# How to Install `urchatbot` on a Slack Workspace

This guide explains how to make `urchatbot` installable by other Slack workspaces and how end users can complete the installation.

## App Identity

- App name: `urchatbot`
- Slack Client ID: `11358846796466.11397955311362`

The Client ID identifies the Slack app during OAuth installation. It is safe to share. Do not share the app's `Client Secret`.

## Who Can Install

`urchatbot` can be installed by:

- A workspace admin or owner
- A member who has permission to install apps
- A member who can submit an install request for admin approval

If the workspace blocks third-party apps, a Slack admin must approve the installation before anyone can use `urchatbot`.

## Before You Share the App

If you own the `urchatbot` Slack app, complete these steps in the Slack app dashboard first.

1. Open [Slack API Apps](https://api.slack.com/apps).
2. Select `urchatbot`.
3. Open `OAuth & Permissions`.
4. Add the required bot scopes for your app.
5. Add at least one valid `Redirect URL`.
6. Open `Manage Distribution`.
7. Complete every required distribution field.
8. Turn on `Public Distribution`.

After public distribution is active, Slack provides an install flow that other workspaces can use.

## Installation Link

Use one of the following options when sharing the app with external users.

### Recommended

Share the public install link generated by Slack after `Public Distribution` is enabled.

This is the best option because Slack manages the correct OAuth parameters for the app.

### Manual OAuth Link

If you need to build the install URL manually, use Slack's OAuth endpoint with the `urchatbot` Client ID and the app's real scopes.

```text
https://slack.com/oauth/v2/authorize?client_id=11358846796466.11397955311362&scope=chat:write,commands,app_mentions:read
```

Update the `scope` list to match the exact scopes configured for `urchatbot`.

If your app uses a specific redirect URL, include it explicitly:

```text
https://slack.com/oauth/v2/authorize?client_id=11358846796466.11397955311362&scope=chat:write,commands,app_mentions:read&redirect_uri=https%3A%2F%2Fyour-domain.example%2Fslack%2Foauth%2Fcallback
```

For production, also include a `state` parameter to protect the OAuth flow from CSRF attacks.

## End User Installation Flow

This is the flow a user follows to install `urchatbot` in their Slack workspace.

1. Open the `Add to Slack` button or direct install link for `urchatbot`.
2. Sign in to Slack if prompted.
3. Choose the workspace where `urchatbot` should be installed.
4. Review the permissions requested by the app.
5. Click `Allow`.
6. Wait for Slack to redirect to the completion page.
7. Return to Slack and open the `urchatbot` app from the Apps section.

If the workspace requires admin approval, Slack may show `Request to Install` instead of `Allow`.

## What Users Should Expect on Each Screen

### 1. Slack Sign-In

The user signs in with the Slack account that has access to the target workspace.

### 2. Workspace Selection

The user picks the workspace where `urchatbot` will be installed.

### 3. Permissions Review

Slack displays the permissions requested by `urchatbot`, such as:

- Sending messages as the bot
- Reading app mentions
- Running slash commands

The exact list depends on how the app was configured in `OAuth & Permissions`.

### 4. Approval

The user clicks `Allow` to continue.

If the workspace restricts app installs, the user requests approval from an admin instead.

### 5. Installation Complete

After approval, Slack redirects to the configured completion page. That page should confirm that `urchatbot` is installed and explain how to start using it.

## First-Time Use After Installation

After the app is installed, users can start with the following steps.

1. Open `urchatbot` from the Slack sidebar.
2. Review the welcome message in App Home or a direct message from the bot.
3. If channel use is supported, invite the bot to a channel.
4. Test one supported command or mention.

Example:

```text
@urchatbot hello
```

If the app supports slash commands, users can also test:

```text
/urchatbot
```

Update these examples to match the real commands implemented by the app.

## Installing `urchatbot` in a Channel

Installing the app in the workspace does not always add it to every channel automatically.

If users want to use `urchatbot` in a channel, they may need to invite it first:

```text
/invite @urchatbot
```

## Troubleshooting

### The install button does not work

- Confirm that the app has `Public Distribution` enabled.
- Confirm that the OAuth scopes are configured.
- Confirm that the redirect URL in Slack matches the app server configuration.

### The user sees `Request to Install`

- The workspace requires admin approval.
- A workspace owner or admin must approve the app before installation completes.

### Installation succeeds but the bot does not respond

- Confirm that the bot was invited to the correct channel.
- Confirm that event subscriptions or slash commands are configured correctly.
- Confirm that the installed workspace token was stored successfully by the app backend.

### The OAuth flow fails after approval

- Confirm that `redirect_uri` matches exactly between the install request and Slack app settings.
- Confirm that the app backend exchanges the OAuth `code` correctly.
- Confirm that the app uses the correct `Client ID` and `Client Secret`.

## Security Notes

- Share the `Client ID` freely if needed.
- Never publish the `Client Secret`.
- Use a `state` parameter in production OAuth flows.
- Request only the minimum scopes required by `urchatbot`.
