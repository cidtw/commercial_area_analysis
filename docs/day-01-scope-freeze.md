# Day 01 Scope Freeze

![Day 01 Scope Freeze Poster](/docs/assets/day-01-scope-freeze-poster.png)

## 목적

주제는 이미 확정되었다는 전제에서, 개발 초안 12일 빌드업의 1일차 목표는 “무엇을 만들지”를 다시 논의하는 것이 아니라 “무엇을 기준으로 이후 11일을 진행할지”를 고정하는 것이다.

## 분석 대상 권역

- 기본 시연 권역: 성동구 성수권
- 우선 행정동:
  - 성수1가1동
  - 성수1가2동
  - 성수2가1동
- 이유:
  - 현재 mock/sample 데이터와 API 응답이 이미 이 권역을 기준으로 구성되어 있다.
  - 홈, 분석 폼, 샘플 결과 화면이 모두 이 권역을 전제로 설명되고 있다.

## 핵심 업종군

- 1차 발표 기준 핵심 업종:
  - 카페
  - 디저트
  - 베이커리
  - 일반음식점
- 보조 업종:
  - 라이프스타일 소매
  - 피트니스
- 운영 원칙:
  - 발표 시연은 `카페`를 기본 시나리오로 삼는다.
  - 다른 업종은 fallback 또는 비교 예시로만 사용한다.

## source metadata 범위

아래 5개 필드는 이후 문서, API, 결과 화면에서 빠지지 않는 기본 범위로 고정한다.

- `source_name`
- `source_version`
- `reference_date`
- `license_note`
- `data_mode`

운영 모드:

- `mock`: UI/테스트 기본값
- `sample`: 실제 연동 구조 검증용 fixture
- `real`: 로컬 import 또는 운영 적재 데이터

## 용어 정리

- 데이터 수집:
  - 원천 데이터, fixture, source metadata를 다루는 단계
- 데이터 취합 및 분석:
  - 좌표, 반경, 경쟁도, 수요, 안정성, 데이터 범위를 같은 분석 계약으로 정리하는 단계
- 워크플로우 구현:
  - 위치 검색, 좌표 확정, 조건 선택, 분석 실행, 결과 조회 흐름을 연결하는 단계
- 시각화 및 가공:
  - 결과를 의사결정용 리포트로 읽히게 만드는 단계

화면 용어:

- 홈:
  - 프로젝트 구조와 현재 상태 소개
- 분석 폼:
  - 위치와 조건 확정
- 결과 화면:
  - 점수보다 근거 우선 확인
- 방법론/출처:
  - 데이터 한계와 계산 기준 보완

## 오늘 완료 기준

- 이 문서의 네 섹션이 모두 채워져 있다.
- 이후 문서가 참조할 기본 권역, 기본 시연 업종, metadata 범위, 핵심 용어가 미정 상태가 아니다.

- 새 용어를 만들기보다 여기 정의된 표현을 우선 사용한다.
