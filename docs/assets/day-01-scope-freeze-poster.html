코드에 익숙하지 않은 사람도 오늘의 결과를 한눈에 이해할 수 있도록, 1일차는 범위를 줄이고 용어를 맞추는 데 집중합니다. 이후 11일은 이 기준을 흔들지 않고 쌓아 올리는 방식으로 진행합니다.

기본 시연 권역: 성수권 핵심 시연 업종: 카페 기본 데이터 모드: mock → sample → real
분석 대상 권역 성수1가1동 · 성수1가2동 · 성수2가1동

현재 데이터와 화면이 이미 이 권역을 기준으로 정렬되어 있습니다.

핵심 업종군 카페 · 디저트 · 베이커리 · 일반음식점

발표 시연은 카페를 기본 시나리오로 고정합니다.

source metadata 범위 name · version · date · license · mode

어떤 숫자든 원천과 기준일을 역으로 설명할 수 있어야 합니다.

오늘의 목표 후속 11일이 흔들리지 않을 기준 만들기

day-02부터는 이 기준을 바꾸지 않고 구조와 화면을 쌓습니다.

01 데이터 수집

원천 데이터, fixture, source metadata를 같은 언어로 관리합니다.

02 데이터 취합 및 분석

좌표, 반경, 경쟁도, 수요, 안정성을 하나의 분석 계약으로 정리합니다.

03 워크플로우 구현

위치 검색 → 조건 선택 → 분석 실행 → 결과 조회 흐름을 끊김 없이 연결합니다.

04 시각화 및 가공

최종 결과를 숫자 나열이 아니라 의사결정용 리포트로 읽히게 만듭니다.

비개발자에게 중요한 포인트

오늘은 기능을 늘리는 날이 아니라, 발표와 개발의 기준선을 맞추는 날입니다.
이 기준이 있어야 내일부터 API 구조, UX 초안, 데모 시나리오가 흔들리지 않습니다.
즉, 1일차의 성과는 “무엇을 안 할지까지 정한 것” 자체입니다.
